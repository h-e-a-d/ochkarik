# Tailwind Pre-Build Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the synchronous Tailwind CDN `<script>` with a pre-built static CSS file generated at build time, eliminating the single biggest render blocker on the site.

**Architecture:** Install `tailwindcss@3` CLI as a devDependency; add a `tailwind.input.css` source file; generate `tailwind.css` as a build artifact before Eleventy runs; replace the CDN `<script>` tag in all three templates with a versioned `<link>` tag; wire the new file into the Service Worker cache.

**Tech Stack:** Tailwind CSS v3 CLI, Eleventy 3, Cloudflare Pages (auto-deploys from GitHub via `npm run build`).

- [ ] **Step 2: Create `tailwind.config.js`**

This must cover all three templates. The `content` globs tell Tailwind which files to scan for class names so unused utilities get purged. The theme merges the two slightly-different configs used across `index.njk` vs blog templates into one canonical config.

Create `/Users/egalvans/Downloads/Head/Claude/ochkarik-main/tailwind.config.js`:

```js
/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [
    './src/**/*.njk',
    './src/**/*.html',
    './src/**/*.md',
    './script.js',
    './vision-test.js',
    './vision-disorders.js',
  ],
  theme: {
    extend: {
      colors: {
        // Both nested (navy.900) and flat (navy-900) class forms resolve to
        // the same utility names (e.g. bg-navy-900). Using the nested form.
        navy: {
          800: '#0f3d56',
          900: '#0a2a3d',
        },
        coral: '#ff6b4a',
      },
      fontFamily: {
        // font-sans override used in src/index.njk body
        sans: ['Helvetica Neue', 'Helvetica', 'Arial', 'sans-serif'],
        // font-poppins used in blog templates body class
        poppins: ['Poppins', 'sans-serif'],
      },
    },
  },
  plugins: [],
};
```

- [ ] **Step 3: Verify config is valid**

```bash
npx tailwindcss --help
```

Expected: Help text printed without errors.

- [ ] **Step 4: Commit**


## Task 3: Update the build pipeline

**Files:**
- Modify: `package.json`
- Modify: `.eleventy.js`

- [ ] **Step 1: Update `package.json` scripts**

Tailwind must run **before** Eleventy so the generated `tailwind.css` exists when Eleventy's passthrough copy executes.

Open `package.json`. Replace:

```json
"scripts": {
    "dev": "eleventy --serve --quiet",
    "build": "eleventy",
    "clean": "rm -rf _site"
  },
```

With:

```json
"scripts": {
    "dev": "tailwindcss -i ./tailwind.input.css -o ./tailwind.css && eleventy --serve --quiet",
    "build": "tailwindcss -i ./tailwind.input.css -o ./tailwind.css --minify && eleventy",
    "clean": "rm -rf _site tailwind.css"
  },
```

Notes:
- `dev` builds Tailwind once (no `--minify`, so class names are readable in DevTools) then starts Eleventy's dev server. Class additions during development require stopping and restarting the dev server, which is acceptable — Tailwind class changes are infrequent.
- `build` (used by Cloudflare Pages) adds `--minify`.
- `clean` also removes the build artifact.

- [ ] **Step 2: Add passthrough copy for `tailwind.css` to `.eleventy.js`**

Open `.eleventy.js`. Add this line immediately after the existing passthrough copy block (after line 8, `eleventyConfig.addPassthroughCopy({ "sw.js": "sw.js" });`):

```js
  eleventyConfig.addPassthroughCopy({ "tailwind.css": "tailwind.css" });
```

Full updated passthrough block for reference:

```js

  eleventyConfig.addPassthroughCopy({ "styles.css": "styles.css" });
  eleventyConfig.addPassthroughCopy({ "script.js": "script.js" });
  eleventyConfig.addPassthroughCopy({ "vision-test.js": "vision-test.js" });
```

## Task 4: Update Service Worker cache

**Files:**
- Modify: `sw.js`

The SW uses a versioned URL for `styles.css` to avoid stale cache issues. Apply the same pattern to `tailwind.css`. Bump `CACHE_VERSION` to `1.3.1` so all existing users get a clean cache install that includes the new file.

- [ ] **Step 1: Edit `sw.js`**

Current top of `sw.js` (lines 1-11):

```js
const CACHE_VERSION = '1.3.0';
const CACHE_NAME = `sitorakarimi-${CACHE_VERSION}`;

// CSS version — must match the ?v= query string on /styles.css in index.njk.
// Update BOTH values whenever styles.css changes, then bump CACHE_VERSION.
const CSS_VERSION = '1.3.0';
```

Replace with:

```js
const CACHE_VERSION = '1.3.1';
const CACHE_NAME = `sitorakarimi-${CACHE_VERSION}`;

// CSS version — must match the ?v= query string on /styles.css in index.njk.
// Update BOTH values whenever styles.css changes, then bump CACHE_VERSION.
const CSS_VERSION = '1.3.0';

// Tailwind version — must match the ?v= query string on /tailwind.css in all templates.
// Update BOTH values whenever tailwind.config.js or tailwind.input.css changes, then bump CACHE_VERSION.
const TAILWIND_VERSION = '1.3.1';
```

- [ ] **Step 2: Add `tailwind.css` to `ASSETS_TO_CACHE` in `sw.js`**

Current `ASSETS_TO_CACHE` starts with:

```js
const ASSETS_TO_CACHE = [
    '/ru/',
    '/tj/',
    '/en/',
    '/script.js',
    '/vision-test.js',
    '/vision-disorders.js',
    `/styles.css?v=${CSS_VERSION}`,
```

Add the Tailwind line immediately after the `styles.css` line:

```js
const ASSETS_TO_CACHE = [
    '/ru/',
    '/tj/',
    '/en/',
    '/script.js',
    '/vision-test.js',
    '/vision-disorders.js',
    `/styles.css?v=${CSS_VERSION}`,
    `/tailwind.css?v=${TAILWIND_VERSION}`,
```

- [ ] **Step 3: Commit**

```bash
git add sw.js
git commit -m "feat: add tailwind.css to SW cache; bump CACHE_VERSION to 1.3.1"
```

## Cloudflare Pages Note

Cloudflare Pages runs `npm run build` using the command set in the Pages project settings. Since `@11ty/eleventy` is already in `devDependencies` and the site deploys successfully, Cloudflare Pages is already installing devDependencies (either because `NODE_ENV` isn't set to `production`, or the install command includes `--include=dev`). Adding `tailwindcss@3` to `devDependencies` follows the same pattern and will work.

**If the Cloudflare build fails** with a "tailwindcss not found" error, the fix is: in the Cloudflare Pages project settings → Build → set the **Install command** to `npm install` (instead of the default `npm ci`), or add `NODE_ENV=development` as an environment variable.
